# 使用文档入口

`.ai/rules` 只负责维护规则、架构约束和实现边界。

如果你要在自己的项目中使用 Vibe Forge，继续看 [docs/index.md](../docs/index.md)。

约定：

- 面向用户的安装、运行、CLI 和插件使用说明统一放在 `.ai/docs/`
- `.ai/rules` 中只保留对这些文档的入口提示，不再直接承载具体使用说明
- 仓库根 README 只保留用户真正关心的关键信息：入口、预览、安装、使用与文档导航；不要在外层 README 堆实现细节、内部运行机制或冗长背景说明
- README 默认使用双语分文件组织：`README.md` 作为英文主入口，中文放在 `README.zh-Hans.md`，并在文件顶部提供互相切换链接
- README 截图优先展示真实的主使用界面，而不是设置页、空白态或边角页面；截图内容应带有适量真实数据，能体现核心交互效果
- GitHub 展示图默认用 `<picture>` 适配明暗色模式，为 light / dark 分别提供产物；不要混用单张截图去兼容所有主题
- 如果项目存在公开的在线入口（例如 PWA / Web UI），应直接在 README 外层显式给出，不要把入口只藏在深层使用文档里
