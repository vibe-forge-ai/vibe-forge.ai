## 目标与分层
- 目标：在当前单体后端+SQLite+前端体系内，补齐会话/历史、知识库、自动化、配置的核心能力，并按“主路径优先、低耦合先行、结构性演进后置”的顺序落地。
- 分层：数据层（SQLite/索引）→ 服务层（API/任务编排）→ 前端体验（展示与编辑）→ 运行可靠性（调度与权限）。

## 排序原则（难度 × 主路径）
- 主路径高 + 难度低：优先推进，用于快速提升体验与验证方向。
- 主路径高 + 难度中/高：拆成阶段，先做“最小可用”再做“结构升级”。
- 主路径低：延后，与主路径改造复用组件/数据结构。

## 会话：分析/历史/新建/配置
### 1) 会话分析：完善数据来源、优化展示效果（主路径高 | 难度中）
- 数据来源：
  - 建立“结构化索引表”或“衍生列”，从 messages JSON 抽取 role、tool、token、latency、attachments、entity/spec 引用等字段（避免全表 JSON 解析）。
  - 增量写入：在事件落库时同步写入索引表；历史补全用离线迁移脚本。
  - 统计维度：token 用量、工具调用分布、错误率、会话时长、实体/规则命中。
- 展示优化：
  - 分析页切换：概览卡片、时间线、工具调用矩阵、实体/规则引用图。
  - 组件层抽象：图表组件复用（避免页面逻辑膨胀）。

### 2) 历史：工具扩展、撤回/修改/分叉/引用/合并转发、命令行模式（主路径高 | 难度高）
- 历史查询能力：
  - API 增加游标分页、时间区间、关键词/标签/状态过滤，必要时引入全文索引（SQLite FTS5）。
  - 前端消息流支持“分页回溯 + 定位跳转”。
- 事件级操作：
  - 撤回/修改：以“消息版本”模型实现（message_versions 表）并记录溯源。
  - 分叉/引用/合并转发：以 parentSessionId + message引用链实现；合并转发作为新 session 事件重放。
  - action：统一为“事件类型 + payload”，保持前后端一致的 schema。
- 工具与运行：
  - 运行特定 tool：在会话输入中携带 tool 指令并经服务端权限校验。
  - 等待用户同意再调用：新增“工具调用待确认”状态，前端交互与 WS 事件支持 approve/deny。
- 命令行模式：
  - CLI 与 Web 共享会话协议；对 command 模式做输入聚合与历史复用。

### 3) 新建会话：以特定流程、实体运行（主路径中 | 难度中）
- 新建会话参数扩展：支持指定 spec/entity/rules/skills 组合；服务端在 sessionCreate 时生成 system prompt 并落库。
- UI 增加“模板”选择：从知识库 spec/entity 直接发起会话。

### 4) 配置：默认模型、运行时（主路径中 | 难度低-中）
- 配置 schema 扩展：general 增加 defaultModel/defaultRuntime 字段；并与 modelServices/adapters 做依赖校验。
- 写入与校验：PATCH config 前进行 schema 校验与错误回显；前端展示依赖提示。

## 知识库：新增/编辑/导入（主路径中 | 难度中）
- 写入方式：
  - 新增 API：创建/更新/删除 .ai 文件（限定 workspace 路径），并触发 loader 刷新。
  - 版本处理：简单版本号/时间戳 + 冲突提示（对比 diff）。
- 导入：
  - 支持从本地上传（zip/markdown）解包到 .ai；校验 frontmatter/schema。
- 编辑体验：
  - 详情页进入编辑器（Markdown + frontmatter form）；保存失败/校验反馈。

## 自动化：任务类型、触发方式（主路径中 | 难度中-高）
### 1) 任务类型：以特定流程/实体运行任务 & 恢复特定会话
- 任务模型扩展：task 可指定 spec/entity/rules/skills；可指定基于已有 session 的“恢复运行”。
- 运行结果结构化：automation_runs 增加 status/error/summary/metrics 字段；提升列表可观测性。

### 2) 触发方式：简化 cron 编辑 + 指定周期触发
- 触发器模型扩展：增加“周期式简化配置”（年/月/周/日+具体日期），服务端转为 cron 或内部 schedule。
- 前端编辑器：可视化表单生成 cron，提供预览下次执行时间。

## 风险与技术债
- messages JSON 与结构化索引双写：需迁移脚本与回滚策略。
- 自动化调度器当前单进程：后续可演进到队列/分布式锁。
- 知识库写入涉及安全边界：严格限定 workspace 根目录。
