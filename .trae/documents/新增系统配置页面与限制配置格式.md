## 需求理解
- 新增“系统配置”页面，包含分类：General、Conversation、ModelServices、Adapters、Plugins、MCP、About。
- 配置文件仅支持 json 或 yaml，移除 ts/js 配置加载与类型扩展依赖。
- 系统配置仅展示读取结果，不提供修改。

## 现状定位
- Web 侧路由目前无系统配置页入口；侧边栏有“设置”按钮但未绑定页面。
- 配置加载核心在 packages/core/src/config.ts，当前支持 ts/js/json/yaml。
- 适配器通过 TS module augmentation 扩展配置类型，示例配置为 .ai.config.ts。

## 改动方案
### 1) 前端新增系统配置页与导航入口
- 新增页面组件（如 apps/web/src/pages/Config.tsx），按分类渲染只读配置视图。
- 在路由中新增路径（如 /config 或 /settings）。
- 侧边栏“设置”按钮跳转至新页面。
- 设计只读展示（表格/折叠面板），不提供编辑控件。

### 2) 后端新增配置读取 API（只读）
- 新增路由（如 /api/config）读取核心配置并返回结构化 JSON。
- 仅透出前端展示所需字段，避免泄露敏感信息。
- 复用 core 的 loadConfig / loadAdapterConfig。

### 3) 移除 TS/JS 配置文件支持
- 在 packages/core/src/config.ts 移除 loadJSConfig 与对 .ts/.js/.cjs/.mjs/.cts/.mts 的查找。
- 调整 loadConfig 仅支持 .ai.config.json / .ai.config.yaml|yml（含 dev/infra 变体）。
- 更新 apps/cli/src/config.ts 与文档中关于 defineConfig/TS 配置的引用。
- 处理项目内 .ai.config.ts 示例与类型扩展引用（替换为 json/yaml 示例）。

### 4) 文档与资源同步
- 更新说明文档，明确支持格式与只读配置页。
- 如有 i18n 文本，补充新页面文案。

## 验证方式
- 运行前端与后端开发模式，打开系统配置页确认可展示。
- 调用 /api/config 确认只读数据返回且结构正确。
- 通过示例 json/yaml 配置验证加载逻辑。
